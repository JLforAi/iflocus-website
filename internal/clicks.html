📖操作步驟
1
先在本頁填好測試設定
填入要測試的網址、次數、可執行時段、每次間隔。若只是確認頁面有打開，不需要填 CSS selector。
2
確認 allowlist 有包含目標網域
例如網址是 https://client.com/page，allowlist 要有 client.com。多個網域用逗號分隔，例如 iflocus.com,www.iflocus.com,client.com。
3
到 GitHub Settings 貼上 Secrets / Variables
開啟 JLforAi/iflocus-website repo，依序進入：
Settings -> Secrets and variables -> Actions
  • 點 Secrets 分頁，新增一筆：Name 填 CLICK_TARGET_URL，Secret 填上方 Secrets 區塊產生的網址，例如 https://iflocus.com/internal/clicks.html。
  • 點 Variables 分頁，照上方 Variables 區塊逐行新增。等號左邊填 Name，等號右邊填 Value。
  • 範例：Name 填 CLICK_QA_ENABLED，Value 填 false；Name 填 CLICK_COUNT_PER_RUN，Value 填 1；Name 填 CLICK_ALLOWED_HOSTS，Value 填 iflocus.com,www.iflocus.com。
  • 第一次測試建議先讓 CLICK_QA_ENABLED=false，用手動測試確認沒問題後，再回 Variables 把 Value 改成 true。
4
到 Actions 手動測試 count=1
開啟 Actions -> iFLocus Clicks QA -> Run workflow，照本頁「手動測試參數」填入。第一次請把 count 改成 1，並確認 enabled=true。
5
檢查執行結果與報告
Workflow 跑完後，打開該次 run 的 Artifacts，下載 clicks-qa-results。裡面會有 CSV 報告、config JSON，若有實際執行也會有截圖。
